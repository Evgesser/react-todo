План: Добавление поддержки единиц измерения (units) для todo-элементов

Цель:
- Разделить и сохранять количество и единицу (например: 1.5 л, 2 кг, 3 шт) как отдельные поля в данных задачи, поддержать распознавание из "умного" ввода и показать селектор/поле рядом с полем "кол-во" в форме.

Шаги:
1) Типы
- Добавить `unit?: string` в `Todo` и `TemplateItem` в `src/types/todo.ts`.

2) Парсер
- Обновить `src/utils/parseSmartInput.ts`, расширить `ParsedInput` с полем `unit?: string`.
- В name-first и qty-first ветках возвращать `unit` отдельно, а `comment` — без единицы.

3) Хуки / состояние формы
- В `src/hooks/useTodos.ts` добавить состояние `unit` и `setUnit`.
- Расширить интерфейсы и `addItem`/`api`-payload, чтобы отправлять `unit` на сервер вместе с `quantity`.

4) UI — форма добавления
- В `src/components/TodoForm.tsx` в ряду с полем `quantity` добавить маленькое поле/селектор для `unit`.
- При парсинге текста (`parseSmartInput`) проставлять `todoActions.setUnit(parsed.unit)`.
- Сброс поля `unit` при успешном добавлении.

5) UI — список
- В `src/components/TodoListItem.tsx` отображать единицу рядом с количеством (например: `1.5 л` / `x2 шт`).

6) Диалог выбора количества (опционально)
- По желанию можно добавить выбор единицы в `QuantityDialog` — отложено, сначала простая версия в форме.

7) Локализация
- Добавить ключи `unit`/`unitLabel` в файлы локалей `src/locales/*.ts`.

8) Бэкэнд/API
- Клиентская сторона (`src/lib/api.ts`) уже отправляет объект; серверная часть должна принимать новое поле `unit` (если сервер отдельно — нужно синхронизировать).

Пояснения и варианты:
- UI: сначала добавляем простое текстовое/select поле. В будущем можно заменить на локализованный селектор (л, мл, кг, г, шт и т.д.) по языку.
- Парсер: при распознавании единицы возвращаем её в `unit` и убираем её из `comment`, чтобы не дублировать.

Текущие ограничения:
- Серверная часть не обнаружена в этом репозитории — если она хранит структуру жестко, потребуется обновить API на сервере.

Дальше: внесу минимальные изменения — добавлю поле в типы, обновлю парсер, добавлю состояние `unit` в `useTodos`, и размещу поле `unit` в `TodoForm` рядом с `quantity`. Затем обновлю отображение в `TodoListItem` и переводы.
